2.5. What should you watch out for?
Before starting pre-labeling, it is important to consider a few things. It is important to implement a quality control process to ensure that the pre-labeling is accurate and reliable.

3.3. How can you implement Active Learning? An example
Active Learning can be a bit more complicated compared to pre-leveling details in model training. The process involves a prioritization step that starts with identifying the unable assets on your machine and creating a key project with them. Before the annotation process, the first prioritization step updates the asset priorities, and the most prioritized assets are labeled first. After labeling, a model is trained to predict the enabled assets, and another prioritization step is run with uncertainty sampling, leveraging the model‚Äôs confidence. The updated priorities for the next batch are determined based on this prioritization, and this process repeats until convergence is achieved. However, it‚Äôs worth noting that Active Learning can be challenging when dealing with complex machine learning tasks.
üëâ Key takeaways ‚Ä¢ Active Learning requires a prioritization step to identify assets and create a key project. ‚Ä¢ Prioritization updates are made before the annotation process and most prioritized assets are labeled first. ‚Ä¢ Uncertainty sampling is used to update priorities for the next batch until convergence, but Active Learning can be challenging for complex machine learning tasks.

5. Programmatic QA
We will now shift our focus from reducing labeling time to ensuring the quality of our model performance, as quality is crucial. Our goal is to maintain quality without spending excessive time. One way to do this is through programmatic QA, which involves programming business rules to automatically detect labeling errors, rather than manually reviewing each label. By doing so, we can direct our attention to the assets with the most significant issues, rather than examining every single one. This approach is beneficial because it saves time and resources.
üëâ Key takeaways ‚Ä¢ Quality is crucial for model performance ‚Ä¢ Programmatic QA saves time by automatically detecting labeling errors ‚Ä¢ Focus on assets with significant issues instead of examining every single one
5.1. Increase data quality with automatic error spotting
To save time and improve scalability, it‚Äôs best to avoid manual review of large datasets. While manual review can be useful for getting a sense of errors and providing feedback to the labeling team, it becomes impractical with datasets containing thousands of documents or complex annotations.
Implementing a programmatic step with real-time feedback can help automate the error correction process. By triggering a check either in real-time or at the end of the day, you can ensure that labeling errors are caught and corrected automatically. This saves time and allows you to focus on reviewing more complex errors.
The type of checks you implement will depend on your specific labeling task and business needs. For example, some clients may implement checks on the number of vertebrates per annotation or do geometry checks to ensure consistency. Other examples include visual checks on manufacturing where you may need to check the size of the cardboard boxes.
üëâ Key takeaways ‚Ä¢ Avoid manual review for large datasets ‚Ä¢ Use programmatic steps with real-time feedback for error correction ‚Ä¢ Implement checks specific to labeling task and business needs
5.2. Understanding SAP‚Äôs audit case study
Let‚Äôs take a closer look at a case study involving SAP and document processing. The company needed to extract information from a large number of invoices, which required complex annotations. To streamline the process, they implemented a workflow where only 5% of the assets were manually reviewed, and the rest were checked automatically using a set of rules. These rules included checks on dates, entity format, and key value pairs, among others. As the manual review identified recurring patterns, new rules were added to improve consistency. To demonstrate the process, we can examine a publicly available example of a programmatic QA on a project. The workflow involved uploading a plugin - a simple Python file triggered each time a labeler labeled an asset. The plugin contained two business rules: one checking the format of z bar and the second one validating the currency field. If a rule was not met, an issue was created automatically, and the asset was flagged for review. Using this approach, the number of errors was reduced, and consistency in labeling improved, ultimately creating a more efficient process.
