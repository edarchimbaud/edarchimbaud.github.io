Function introduction
Features
As a set of Python-based quantitative trading program development framework, VeighNa is dedicated to providing quantitative solutions from trading API to strategy auto-trading.
Targeted Users
Try VeighNa if you need to.
Develop your own quantitative trading program based on the Python language to take full advantage of the Python community's powerful data research and machine learning ecology
A standardized trading platform system to connect to many different types of financial markets at home and abroad: securities, futures, options, foreign exchange, etc.
Use a well-tested quantitative strategy engine to complete the entire business process from data maintenance, strategy development, backtesting research to live automated trading
Customize and extend the platform to meet individual trading needs: add trading interfaces, modify the GUI graphical interface, and develop complex strategy applications based on event-driven engines
Control the source code details of the trading program to eliminate all kinds of program backdoors and avoid the risk of strategy theft, interception of trading signals, account password theft, etc.
Save money on quantitative trading platforms, no need to spend tens of thousands of dollars per year on software licensing fees or additional points per transaction
Application Scenarios
Application scenarios for VeighNa can be found from professional individual investors, entrepreneurial private equity, to brokerage capital management departments.
Professional individual investors: use VeighNa Trader to connect directly to the CTP futures desk of futures companies to realize the CTA business process from strategy development to live automated trading
Entrepreneurial private equity: build a server-side unified reporting channel based on RpcService, allowing traders to develop various trading strategy applications on their own local computers
Brokerage capital management department: docking to the securities company's unified deployment of O32 capital management system, custom development of multi-strategy complex system based on event-driven engine

Supported applications
vnpy.app, out-of-the-box trading applications for various quantitative strategies.
cta_strategy: CTA strategy engine module, which allows users to perform fine-grained control of delegated reporting and withdrawal behavior during the operation of CTA-type strategies while maintaining ease of use (reducing trading slippage, implementing high-frequency strategies)
cta_backtester: CTA strategy backtester module, no need to use Jupyter Notebook, directly use the graphical interface to directly carry out strategy backtester analysis, parameter optimization and other related work
spread_trading: multi-contract spread arbitrage module, which not only allows users to start algorithmic trading spreads manually, but also supports users to use the strategy template SpreadStrategyTemplate to develop various spread-based quantitative trading strategies
algo_trading: Algorithmic trading module, providing a variety of commonly used intelligent trading algorithms: TWAP, Sniper, Iceberg, BestLimit, etc. Support commonly used algorithm configuration save
option_master: options volatility trading module, providing volatility curve charts, allowing users to make appropriate judgment analysis, and then use the volatility management component to set the pricing reference volatility, and then you can automatically scan the market for trading opportunities through the options e-eye algorithm and complete the trade instantly
portfolio_strategy: multi-contract portfolio strategy module, designed for quantitative strategies that need to trade multiple contracts simultaneously to meet their historical data backtesting and live automated trading needs
script_trader: script strategy module, designed for multi-standard portfolio trading strategies, also can be directly in the command line to achieve REPL instructions in the form of trading, does not support backtesting function
chart_wizard: Real-time K-line chart module, can realize simple real-time K-line quotes display, directly enter vt_symbol in the edit box of the local contract code, click the [New Chart] button will open the corresponding contract chart
rpc_service: RPC service module, allowing a VeighNa Trader process to be started as a server, as a unified routing channel for quotes and trades, allowing multiple clients to connect at the same time, realizing a multi-process distributed system
excel_rtd: EXCEL RTD module, RTD full name is RealTimeData, is Microsoft mainly for the financial industry's real-time data needs designed for the Excel data interfacing program. The module is used to achieve the function of accessing any data information within the VeighNa program in Excel
data_manager: Historical data management module, which is a multi-functional management tool for historical data within VeighNa Trader. It supports data import, data view and data export, and supports custom data table header format.
data_recorder: the data recorder module, based on the graphical interface for configuration, according to the demand for real-time recording Tick or K-line quotes to the database, for strategy backtesting or live initialization
risk_manager: risk management module, providing statistics and restrictions on rules including trade flow control, number of orders placed, active orders, total number of cancelled orders, etc., effectively realizing front-end risk control functions
web_trader: Web service module, designed for B-S architecture requirements, realizing a Web server that provides active function calls (REST) and passive data push (Websocket)
portfolio_manager: portfolio management module, this module is mainly for various investors who use fundamental strategies, for each investment strategy, create an independent portfolio strategy object
paper_account: simulated trading account module, is to solve the current problem of all kinds of need to rely on server-side functionality of the simulation trading account, directly within the trading client to provide a set of localized simulation trading environment, while based on the real market data of the market to commission aggregation
Generic class components
vnpy.event, a simple and easy-to-use event-driven engine that serves as the core of an event-driven trading program.
vnpy.chart, Python high-performance K-line charts, supports large data volume chart display and real-time data update functions.
vnpy.trader.database, which integrates several database manager modules to support database read and write performance and new database extensions in the future.
vnpy.trader.datafeed, provides standardized interface BaseDataFeed, which brings more flexible data service support.
