Purged k-fold cross validation
Cross-validation (CV) is a widely used technique in machine learning to determine the generalization error of an algorithm, which helps prevent overfitting. The goal of CV is to learn the general structure of the data, so that accurate predictions can be made on future unseen data. When ML algorithms are tested on the same dataset that was used for training, the results may appear to be spectacular. However, this approach is prone to overfitting, as the algorithm can simply memorize the training data without being able to generalize to new data.
CV addresses this issue by splitting the dataset into two sets: the training set and the testing set. The ML algorithm is trained on the training set and then tested on the testing set, which consists of unseen data. The purpose of this process is to simulate the real-world scenario where the algorithm needs to make predictions on new, unseen data. The performance of the algorithm is evaluated based on how well it can predict the outcomes of the testing set.
k-fold cross validation
One of the most popular CV techniques is k-fold CV, where the dataset is divided into k equal-sized subsets, or folds. In each iteration, one of the k subsets is used for testing, and the remaining k-1 subsets are used for training the algorithm. This process is repeated k times, with each of the k subsets used once for testing. The outcome of k-fold CV is a kx1 array of cross-validated performance metrics. For example, in a binary classifier, the model is deemed to have learned something if the cross-validated accuracy is over 1/2, since that is the accuracy we would achieve by tossing a fair coin.
However, standard CV techniques fail when applied to financial problems. This is because financial data is non-stationary and non-IID, which means that the data is not drawn from an independent and identically distributed process. Additionally, the low signal-to-noise ratio in financial data makes it easy to overfit the ML algorithm, even with CV. As a result, it is necessary to modify CV techniques to address the unique challenges posed by financial data.
purged k-fold cross validation
One such modification is purged k-fold CV, which involves purging and embargo techniques to prevent leakage of information from the testing set to the training set.
Purging: Purging involves removing observations from the training set whose labels overlap in time with those in the testing set.
Embargo: Embargo involves removing observations that immediately follow those in the testing set.
These techniques help reduce the likelihood of leakage and improve the accuracy of the ML algorithm in financial applications.
