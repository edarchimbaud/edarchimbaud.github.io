Data API
This API provides stock data using the same API as AlphaVantage, but completely wrapped around the yfinance Python library. The API supports the following functions:
TIME_SERIES_DAILY: Get daily stock data for a given symbol.
NEWS_SENTIMENT: Get news sentiment data for a list of tickers.
OVERVIEW: Get overview data for a given symbol.
INCOME_STATEMENT: Get income statement data for a given symbol.
BALANCE_SHEET: Get balance sheet data for a given symbol.
CASH_FLOW: Get cash flow data for a given symbol.
EARNINGS: Get earnings data for a given symbol.
EARNINGS_CALENDAR: Get earnings calendar data for a given time horizon.
You can access the endpoint at https://data.edarchimbaud.com/query?function=TIME_SERIES_DAILY&symbol=IBM to get daily stock data for IBM, or at https://data.edarchimbaud.com/query?function=NEWS_SENTIMENT&tickers=AAPL to get news sentiment data for Apple.
Endpoints
/query
Parameters
function (required): The API function to use, which can be set to any of the following: TIME_SERIES_DAILY, NEWS_SENTIMENT, OVERVIEW, INCOME_STATEMENT, BALANCE_SHEET, CASH_FLOW, EARNINGS, EARNINGS_CALENDAR.
symbol (required for some functions): The stock symbol to get data for.
tickers (required for NEWS_SENTIMENT): A comma-separated list of stock symbols to get news sentiment data for.
horizon (required for EARNINGS_CALENDAR): The time horizon for earnings data, which can be set to 1day, 1week, 1month, or 3month.
Response
meta: An object containing the metadata of the query, including the function and any required parameters.
data: An array of objects containing the stock data.
Examples
To get daily stock data for IBM:
Bash
Copy
GET /query?function=TIME_SERIES_DAILY&symbol=IBM
Response:
JSON
Copy
{ "meta": { "function": "TIME_SERIES_DAILY", "symbol": "IBM" }, "data": [ { "date": "1986-03-17T00:00:00.000Z", "open": 0.05696210235333443, "high": 0.05717480525350528, "low": 0.05696210235333443, "close": 0.05696210235333443, "volume": 367200 }, ... ] }
To get news sentiment data for Apple and Microsoft:
JSON
Copy
GET /query?function=NEWS_SENTIMENT&tickers=AAPL,MSFT
Response:
JSON
Copy
{ "meta": { "function": "NEWS_SENTIMENT", "tickers": "AAPL,MSFT" }, "data": [ { "title": "Apple wants to sell the iPad as a work device", "sentiment": 0.0014, "symbol": "AAPL" }, { "title": "Microsoft admits it 'misread the future' and kills off Windows 10X", "sentiment": 0.0015, "symbol": "MSFT" }, ... ] }
To get earnings calendar data for the next 3 months:
JSON
Copy
GET /query?function=EARNINGS_CALENDAR&horizon=3month
Response:
JSON
Copy
{ "meta": { "function": "EARNINGS_CALENDAR", "horizon": "3month" }, "data": [ { "date": "2021-04-19", "symbol": "CMA", "epsEstimate": 1.779, "epsActual": null, "epsDifference": null, "surprisePercent": null }, { "date": "2021-04-19", "symbol": "FMBI", "epsEstimate": 0.49, "epsActual": null, "epsDifference": null, "surprisePercent": null }, ... ] }
Error Responses
If any required parameter is missing or invalid, the API will return an error response with a status code of 400 and an error message in the body of the response. For example:
Request:
Bash
Copy
GET /query?function=TIME_SERIES_DAILY
Response:
JSON
Copy
{ "error": "Missing symbol parameter." }
If an unsupported function is requested, the API will return an error response with a status code of 404 and an error message in the body of the response. For example:
Request:
Bash
Copy
GET /query?function=INVALID_FUNCTION&symbol=IBM
Response:
JSON
Copy
{ "error": "Function not supported. Please use one of the following functions: TIME_SERIES_DAILY, NEWS_SENTIMENT, OVERVIEW, INCOME_STATEMENT, BALANCE_SHEET, CASH_FLOW, EARNINGS, EARNINGS_CALENDAR." }
Rate Limiting
The API is rate limited to prevent abuse. Each API key is limited to 5 requests per minute. If you exceed the rate limit, the API will return a 429 status code and an error message in the body of the response. For example:
Request:
Bash
Copy
GET /query?function=TIME_SERIES_DAILY&symbol=IBM&apikey=demo
Response:
JSON
Copy
{ "error": "Rate limit exceeded. Please wait a minute before trying again." }
Authentication
This API does not require authentication, but you can use an API key to access it. To use an API key, simply add the apikey parameter to your requests. For example:
Bash
Copy
GET /query?function=TIME_SERIES_DAILY&symbol=IBM&apikey=your-api-key
